指定ページのUI/UXを「コンポーネント単位」と「ページ全体」の2軸で評価し、改善提案する。

## STEP 1: コンポーネント評価

ページ内の主要コンポーネントを個別に採点する（各10点満点）。

| 観点 | 見るポイント |
|---|---|
| 配色・コントラスト | テキストの読みやすさ、ブランドカラー準拠、背景との対比 |
| 余白・サイズ感 | 内側/外側のpadding/margin、タップ領域の十分さ |
| タイポグラフィ | フォントサイズ・太さの適切さ、行間、文字の切れ |
| 状態の表現 | hover/focus/active/disabled/loading/empty の見た目 |
| アクセシビリティ | alt、aria、フォーカス可視、色だけに頼らない |

出力例:
```
[コンポーネント] CTAボタン「参加申し込み」
  配色:    6/10 — 背景と同系色で埋もれる
  余白:    8/10 — OK
  文字:    9/10 — OK
  状態:    3/10 — hoverもdisabledも変化なし
  a11y:    7/10 — フォーカスリング出る
  小計: 33/50

[コンポーネント] イベントカード
  配色:    8/10 — OK
  ...
```

## STEP 2: ページ全体評価

コンポーネントの良し悪しとは別に、ページ全体を俯瞰して採点する（各10点満点）。

| 観点 | 見るポイント |
|---|---|
| 情報の優先度 | 一番伝えたい情報が最初に目に入るか、視線の流れ |
| ユーザー導線 | 次にやるべきことが迷わずわかるか、CTAまでの距離 |
| レイアウトバランス | セクション間の比重、空白の偏り、全体のリズム |
| 一貫性 | 同じ役割の要素が同じ見た目か、サイト全体との統一 |
| レスポンシブ | モバイル/タブレット/デスクトップで破綻しないか |

出力例:
```
[ページ全体] EventDetailPage
  情報の優先度:      6/10 — 開催日時がファーストビューに入らない
  ユーザー導線:      5/10 — 申込ボタンまでスクロール3回
  レイアウトバランス: 7/10 — 右カラムが寂しい
  一貫性:           8/10 — 概ね統一
  レスポンシブ:      4/10 — SP時にテーブルがはみ出す
  小計: 30/50
```

## STEP 3: 総合スコアと改善提案

```
━━━━━━━━━━━━━━━━━━━━━━━
総合: コンポーネント平均 XX/50 + ページ全体 XX/50 = XX/100
━━━━━━━━━━━━━━━━━━━━━━━
```

1. スコアが最も低い項目を3つピックアップ
2. それぞれ具体的な改善案をコード付きで提案
3. ユーザーの指示があれば修正を実行

## 手順
1. 対象ページのコードを読む（引数でページ名指定、なければ前回最低スコアまたは未評価のページ）
2. STEP 1 → STEP 2 → STEP 3 の順に実行
3. 改善を実施したら再採点して差分を見せる

## ループで使う場合
`/loop 10m /design-review` で定期的にページを順番にレビュー→改善を繰り返す。
